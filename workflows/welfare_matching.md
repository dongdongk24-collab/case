# Workflow: 복지서비스 매칭 분석

## 목표
민원인 정보와 복지 자료 PDF를 입력받아 Gemini AI가 6단계 사례관리 보고서를 생성한다.
광진구 지역 서비스를 우선으로, 서울시·전국 서비스까지 실시간 웹 검색으로 찾아 매칭한다.

## 입력값
- 민원인 기본 정보: 연령, 가구구성, 소득수준, 주거형태, 건강상태, 특이사항
- 복지 자료 PDF (선택): 광진구 복지안내서, 각종 사업안내 문서 등

## 단계
1. `streamlit run app.py` 실행 → 웹 화면 열림
2. 사이드바에 민원인 정보 입력
3. PDF 파일 업로드 (선택) → `tools/pdf_reader.py`가 텍스트 추출
4. "복지서비스 매칭 시작" 클릭
5. `tools/gemini_client.py` 호출 → Gemini AI + Google 웹 검색 실행
6. 6단계 보고서가 화면에 출력됨

## 결과물
- 1단계: 민원인 상황 요약
- 2단계: 욕구 분석
- 3단계: SMART 사례관리 목표
- 4단계: 복지 서비스 매칭 표 (출처 포함)
- 5단계: 개입 방향 및 실현 가능성 평가
- 6단계: 민원인 설명용 문안

## 문제 대응
- **PDF 추출 실패**: 해당 파일 건너뛰고 경고 표시, 나머지 진행
- **API 오류 (키 문제)**: `.env` 파일에 `GEMINI_API_KEY` 확인
- **웹 검색 오류**: 자동으로 검색 없이 재시도 (학습 데이터 기반)
- **응답 없음**: 30초~1분 기다린 후 재시도
- **토큰 초과**: PDF가 너무 크면 자동으로 앞부분만 사용 (60,000자 제한)
